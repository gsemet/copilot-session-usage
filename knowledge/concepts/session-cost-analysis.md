# Session Cost Analysis

## Overview

Session cost analysis is the process of extracting token usage metrics from Copilot debug logs and estimating the monetary cost based on per-model pricing.

## Key Concepts

### Token Types

- **Input tokens** — Tokens sent to the model (prompt + context)
- **Output tokens** — Tokens generated by the model (response)
- **Cached tokens** — Input tokens that hit the cache (discounted rate)

### Pricing Tiers

Models may have multiple pricing tiers based on context window size:

- **Default tier** — Standard pricing for typical context lengths
- **Long context tier** — Higher pricing for very long contexts (e.g. > 272k tokens)

### Cost Attribution

Costs can be attributed at different granularities:

- **Session-level** — Total cost for the entire session
- **Model-level** — Cost broken down by each model used
- **Subagent-level** — Cost attributed to individual subagent calls

### Cache Ratio

The cache ratio measures efficiency:

```
cache_ratio = cached_tokens / input_tokens
```

A high cache ratio (> 0.5) indicates good cache utilization and lower effective cost.
