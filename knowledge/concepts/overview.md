# Session Cost Tracking Overview

## Core Principle

Every LLM request made by a coding agent (parent or subagent) is logged as a structured event in JSONL files. These events contain input tokens, output tokens, cached tokens, model name, and timestamps. The VS Code Copilot extension aggregates these events to produce the "Agent Debug Logs" panel metrics.

## What Gets Tracked

| Metric | Source Field | Description |
|--------|-------------|-------------|
| Input Tokens | `attrs.inputTokens` | Tokens sent to the model (prompt + context) |
| Output Tokens | `attrs.outputTokens` | Tokens generated by the model |
| Cached Tokens | `attrs.cachedTokens` | Prompt tokens served from cache (not billed) |
| LLM Calls | Count of `llm_request` events | Number of individual model invocations |
| Model | `attrs.model` | Which model handled the request |

## Where Data Lives

Session logs are stored per-workspace in:

```
~/Library/Application Support/Code/User/workspaceStorage/<ws-id>/GitHub.copilot-chat/debug-logs/<session-id>/
```

Each session directory contains:
- `main.jsonl` — Parent agent events
- `runSubagent-<name>-functions.runSubagent:<id>.jsonl` — Per-subagent events
- `title-*.jsonl` — Title generation events (usually 1 call)
- `models.json`, `tools_*.json`, `system_prompt_*.json` — Metadata (no token events)

## Key Insight for Agents

> The debug panel's "Total Input Tokens" is the SUM of `inputTokens` across **all `.jsonl` files** in the session directory. Do not look only at `main.jsonl` — subagents often account for 70-80% of total cost.
